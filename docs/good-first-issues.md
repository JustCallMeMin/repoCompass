# Good First Issues Guide

Welcome to the RepoCompass project! We use the `good first issue` label to mark tasks that are scoped specifically for new contributors.

## Characteristics of a Good First Issue
- The problem is well-defined and isolated.
- The solution does not require deep knowledge of the core engine.
- Often involves adding an Analyzer, expanding a fixture test, or fixing a typo.

## How to Claim an Issue
1. Find an issue labeled `good first issue`.
2. Leave a comment: "I would like to work on this."
3. A maintainer will assign it to you.
4. If you get stuck, ask questions in the issue! We are happy to help.
