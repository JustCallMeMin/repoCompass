# Maintainer Triage Guide

This guide is for RepoCompass maintainers to ensure a healthy and responsive community.

## Daily Triage Routine
1. Review issues labeled `needs-triage`.
2. Apply appropriate type and ecosystem labels.
3. If an issue lacks detail, add the `needs-information` label and ask the author for clarification.
4. If an issue is well-scoped and easy, add the `good first issue` label.
5. Remove `needs-triage` once categorized.

## PR Review Standards
- Ensure the PR template is filled out.
- Ensure all CI checks pass.
- If golden files were updated, review the diff carefully.
- Merge using "Squash and merge" for clean history on `main`.
