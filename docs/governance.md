# Governance

RepoCompass is an open-source project driven by community contribution.

## Maintainership

At this stage, RepoCompass uses a **Benevolent Dictator For Life (BDFL)** model. The original creator has final say in architectural decisions and merge rights to `main`.

As the project grows, we aim to transition to a distributed maintainership model.

## Making Decisions

1. **Bug Fixes**: Reviewed and merged as long as they pass CI and tests.
2. **New Analyzers**: Discussed in an issue first. We prefer adding generic, widely applicable analyzers over niche ones.
3. **Core Architecture Changes**: Require an extensive discussion on GitHub Discussions and an approved architecture proposal document.
